# 1Panel App Store

一个用于集中管理、同步与分发 **1Panel 应用** 的资源仓库。

## 📖 仓库概览

本仓库旨在为 **1Panel** 用户与开发者提供标准化的应用资源管理方案。为了兼顾“自动化同步”与“快速手动安装”两种使用场景，我们将资源划分为两个核心目录：

-  **​`/apps`​**：存放应用原始目录结构，适合自建仓库、自动化同步、持续集成及开发调试。
-  **​`/apps-zip`​**：存放打包好的 `.zip` 安装包，适合下载后在 1Panel 面板中直接导入。

## 🛠 使用指南

### 方式一：自建应用仓库 (推荐开发者/运维)

如果你需要持续追踪应用更新或批量同步：

1. 使用 `/apps` 目录作为你的 1Panel 本地仓库源。
2. 配置 1Panel 的“应用商店”地址指向本仓库（或通过脚本自动拉取）。
3. **优势**：目录结构完整，版本控制方便，支持自动化部署。

### 方式二：手动导入 (推荐普通用户)

如果你只需要快速部署单个应用：

1. 下载对应的 `.zip` 文件至本地。
2. 登录 1Panel 面板 -\> **应用商店** -\> **导入应用**。
3. 上传下载的压缩包，按页面提示完成配置即可。
4. **优势**：简单快捷，无需配置环境。

## 📝 维护规范

为保持仓库整洁统一，建议新增应用时遵循以下准则：

1. **命名一致性**：`apps/` 中的目录名称与 `apps-zip/` 中的 ZIP 文件名应保持一致。
2. **列表同步**：新增应用后，请务必更新 README 中的“应用列表”表格。
3. **元数据补充**：建议在应用根目录完善 `metadata.json`，并提供清晰的说明文档。

## 💡 常见问题 (FAQ)

**Q：为什么同时保留** **​`apps`​** **和** **​`apps-zip`​**​ **？**

A：两者面向场景不同。`apps` 侧重于代码版本管理与自动同步；`apps-zip` 侧重于分发与离线部署，兼顾了开发者与普通用户。

**Q：普通用户应该选择哪个？**

A：建议直接使用 `apps-zip/` 目录中的压缩包进行导入。
